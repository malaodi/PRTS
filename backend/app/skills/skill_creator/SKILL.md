# skill-creator

## 概述
此技能指导 Agent 如何将对话中学到的知识、方法、工作流程沉淀为 Skill（SKILL.md）资产。

## 触发时机
当用户在对话中表达"将某内容保存为技能"、"把这个做法沉淀下来"等意图时，Agent 应加载此技能执行资产生成。

## 执行流程

### 1. 识别可沉淀内容
从当前对话上下文中提取：
- 刚完成的工作流程 / 最佳实践 / 方法论
- 用户明确说"记住这个做法"的片段
- Agent 分析出的可复用知识

### 2. 生成 SKILL.md 内容
SKILL.md 为 YAML frontmatter + Markdown 正文：

```markdown
---
trigger_keywords:
  - "关键词1"
  - "关键词2"
description: "技能功能的简要说明"
version: "1.0.0"
author: "{从对话上下文推断}"
---

# {技能标题}

## 概述
{一段话说明这个技能做什么、什么时候用}

## 适用场景
{列举适用场景}

## 执行步骤
{结构化步骤，Agent 可以逐步执行}

## 注意事项
{使用中的约束、常见陷阱}

## 参考
{相关链接、文档}
```

### 3. 必须调用 create_asset 工具
生成 SKILL.md 后，必须调用 `create_asset` 工具，参数：
- asset_type: "skill"
- name: 技能名称
- description: 总结描述
- content: SKILL.md 完整内容
- visibility: "private" (默认)
- tags: 相关标签列表

create_asset 工具会弹出确认卡片给用户，用户确认后写入空间。

### 4. 命名原则
- 用动词开头或功能名称：`数据分析与可视化`、`PPTX生成`
- 短小精悍，3-8个汉字
- 避免"技能"、"skill"等冗余后缀

## 示例

用户说："记住，以后做财报分析要先用 read_document 解析 PDF，然后提取资产负债表和利润表的关键数据，再做同比环比计算"

Agent 应：
1. 加载 skill-creator SKILL.md
2. 生成 SKILL.md：

```markdown
---
trigger_keywords:
  - "财报分析"
  - "财务分析"
  - "分析财报"
  - "财务报表"
description: "财务报告分析标准流程，支持PDF解析和同比环比计算"
version: "1.0.0"
---

# 财报自动化分析

## 概述
标准化的财务报告分析流程，从PDF解析到数据计算，适用于年报/季报分析。

## 适用场景
- 上市公司年报/季报分析
- 内部财务报表审查
- 财务数据同比环比对比

## 执行步骤
1. 使用 `read_document` 解析财务报表 PDF
2. 提取资产负债表关键数据（总资产、负债、净资产）
3. 提取利润表关键数据（营收、净利润、毛利）
4. 计算同比（YoY）和环比（QoQ）变化率
5. 生成分析报告，突出异常变动项

## 注意事项
- 仅适用于标准财务报表格式
- 非标格式需先预览确认结构
- 数据单位需统一（万元/亿元）

## 参考
- 财务报表分析标准指标
```

3. 调用 create_asset(type="skill", ...) 弹出确认卡片
